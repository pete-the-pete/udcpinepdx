# Chef CSS Effects (jig + heat) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Give the pizza chef per-state CSS motion — a happy jig in the
`active` state, and a pulsing red glow plus rising steam in the `hot` state.

**Architecture:** Uses the engine's existing `css_animation` mechanism. The
manifest declares a `css_animation` name per state; the engine owns a
catalogue mapping each name to what it does. The manifest is generated by
`design/chef/scripts/pack.py`, so the durable declaration lives in that
script's `STATES` table; the generated `chef.manifest.json` is hand-edited
to match (re-running the pipeline would re-pack sprite sheets through an ML
background-remover — an unwanted side effect of a metadata change).

**Tech Stack:** CSS `@keyframes`, Preact, the Python `pack.py` art pipeline.
No new dependencies.

## Context

The chef engine (`plans/web/2026-05-17-pizza-chef-engine.md`, Decision 4)
established: the manifest declares which states want a named CSS animation,
and the engine defines what the names mean. Today the catalogue has one
entry — `shiver` (on `frozen`). This plan adds two more:

- `jig` — `active`'s animation. A 4-beat hop: lean onto one foot, down,
  lean onto the other foot, down. `rotateY` for the one-foot lean,
  `translateY` for the bounce.
- `heat` — `hot`'s effect. Two parts: a pulsing red `drop-shadow` glow on
  the sprite, and three rising-and-fading steam wisps rendered behind it.

`heat` is the first catalogue entry that does more than animate the sprite
(it adds decoration elements), so the catalogue value becomes a small object
(`{ sprite, steam? }`) rather than a bare animation string.

Sprite sheets are all 1-frame today; these effects are pure CSS on top of
the static sprite and are independent of the frame-cycle mechanism.

## File structure

- `design/chef/scripts/pack.py` — `STATES` table: the durable source of
  truth for each state's `css_animation`. Modified.
- `web/frontend/src/assets/chef/chef.manifest.json` — the generated
  artifact. Hand-edited to match `pack.py`.
- `web/frontend/src/chef/chef.css` — the keyframe catalogue (engine-owned).
  Gains `chef-jig`, `chef-hot-glow`, `chef-steam`, the steam element styles,
  and a small DOM-structure change (a `.chef__stage` wrapper) so steam is
  positioned relative to the sprite in both widget modes. Modified.
- `web/frontend/src/chef/ChefWidget.tsx` — `ChefSprite` gains the
  `CHEF_EFFECTS` catalogue and renders the steam wisps. Modified.

The four files must agree on the two `css_animation` names — `"jig"` and
`"heat"` — exactly.

## Why a `.chef__stage` wrapper

Today `ChefSprite` renders a single `.chef__sprite` div sized directly. The
steam wisps must sit in a box that matches the *sprite's* size — but in
expanded mode the `.chef` container is a full-screen overlay, not the
sprite's box. So `ChefSprite` now renders a `.chef__stage` wrapper sized to
the sprite (200×200 compact, `min(70vw,70vh)` expanded); the sprite and the
steam layer are absolutely-positioned children that fill it. The sprite's
jig `transform` moves only the sprite layer, so steam rises independently.

- [ ] **Step 1: Add the `CHEF_EFFECTS` catalogue**

In `web/frontend/src/chef/ChefWidget.tsx`, immediately before the

```tsx
/**
 * The css_animation catalogue. The manifest declares a name per state; this
 * map defines what each name means — a CSS animation on the sprite, and
 * optionally decoration (steam wisps). Transform (shiver/jig), filter (glow),
 * and background-position (frame cycle) are distinct properties, so a state's
 * sprite animation always composes with the cycle without conflict.
 */
interface ChefEffect {
  sprite: string;
  steam?: boolean;
}

const CHEF_EFFECTS: Record<string, ChefEffect> = {
  shiver: { sprite: "chef-shiver 0.18s ease-in-out infinite" },
  jig: { sprite: "chef-jig 0.72s ease-in-out infinite" },
  heat: { sprite: "chef-hot-glow 1.3s ease-in-out infinite", steam: true },
};
```

- [ ] **Step 2: Rewrite `ChefSprite` to use the catalogue and render steam**

Replace the entire `ChefSprite` function — currently:

```tsx
function ChefSprite({ state }: { state: ChefState }) {
  const spec = manifest.states[state];
  const url = sheetFor(state);
  const style: JSX.CSSProperties = {
    backgroundImage: url ? `url("${url}")` : "none",
  };

  const animations: string[] = [];
  if (spec.frames > 1) {
    const fps = spec.fps ?? 8;
    style.backgroundSize = `${spec.frames * 100}% 100%`;
    animations.push(

    style.backgroundSize = "100% 100%";

  if (spec.css_animation === "shiver") {
    animations.push("chef-shiver 0.18s ease-in-out infinite");
  }
  if (animations.length > 0) style.animation = animations.join(", ");

  return <div class="chef__sprite" style={style} aria-hidden="true" />;
}
```

— with:

```tsx
function ChefSprite({ state }: { state: ChefState }) {
  const spec = manifest.states[state];
  const effect = spec.css_animation
    ? CHEF_EFFECTS[spec.css_animation]
    : undefined;
  const url = sheetFor(state);
  const style: JSX.CSSProperties = {
    backgroundImage: url ? `url("${url}")` : "none",
  };

  const animations: string[] = [];
  if (spec.frames > 1) {
    const fps = spec.fps ?? 8;
    style.backgroundSize = `${spec.frames * 100}% 100%`;
    animations.push(
      `chef-cycle-${state} ${spec.frames / fps}s steps(${spec.frames}) infinite`,
    );
  } else {
    style.backgroundSize = "100% 100%";
  }
  if (effect) animations.push(effect.sprite);
  if (animations.length > 0) style.animation = animations.join(", ");

  return (
    <div class="chef__stage">
      {effect?.steam && (
        <div class="chef__steam" aria-hidden="true">
          <span class="chef__wisp" />
          <span class="chef__wisp" />
          <span class="chef__wisp" />
        </div>
      )}
      <div class="chef__sprite" style={style} aria-hidden="true" />
    </div>
  );
}
```

The steam layer is rendered *before* the sprite so it paints behind it; the
sprite's transparent regions let the wisps show through and rise above the
chef.
